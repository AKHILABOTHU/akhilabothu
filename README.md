<p dir="auto">I love voice UIs <g-emoji class="g-emoji" alias="purple_heart" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/1f49c.png">💜</g-emoji>. I believe that voice will only become more prominent over time. However, today's voice UI and speech sdks are vendor-dependent and provide more an interface to platform APIs than tools to build sophisticated UIs. Amazon developers build lambda functions and use the Alexa Console while Google Assistant developers develop on Dialogflow or Actions on Google and use different sdks. react-ssml-dom is a research project that aims to take advantage of React and its component-based UI development for voice UIs. It also provides a middleware engine to parse any request from any vendor to a unified runtime payload and then revert back the response to the vendor-based response JSON. You can find the package <a href="https://www.npmjs.com/package/react-ssml-dom" rel="nofollow">here on npm</a>. I wrote more about my motivation <a href="https://andre-timo-landgraf.medium.com/a-react-renderer-for-ssml-91cdd1d66b3e" rel="nofollow">here on Medium.com</a>. <g-emoji class="g-emoji" alias="eyes" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/1f440.png">👀</g-emoji></p>
